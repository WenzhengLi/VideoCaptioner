# TASK-017：Dify“阿峰”应用与最终验收

## 状态

待执行；依赖 TASK-016。

## 目标

在已通过检索验收的正式 Dataset 上创建“阿峰” Workflow/Chatflow，实现课程方法复现回答、证据引用、
多方案组织和可复制表达，同时保持课程忠实度边界。

## 必须完成

1. 应用名称：`阿峰`；Dataset：正式高质量 Dataset；
2. 固定角色：课程方法复现助手，不使用外部知识纠正课程；
3. 输出区分：课程原话、课程总结、课程方法直接改写、证据不足；
4. 每个主要判断必须包含 knowledge ID、课程/案例、时间范围和 evidence IDs；
5. `partial_method` 必须显式显示课程覆盖不完整；
6. 课程未覆盖时返回证据不足；
7. manual_review/rejected 不得被检索或生成；
8. 设计至少 20 个应用级问题，覆盖单课问答、跨课比较和用户情境建议；
9. 检查无证据推断、引用错配、模型补写和可复制话术来源；
10. 导出 Workflow DSL、全部 Prompt、变量、节点顺序和验收报告。

## 外部依赖

- Dify 已配置合法 LLM 供应商；
- 不得把 CC Switch 的本地 Token 自动复制到 Dify；
- 缺少 Dify LLM 配置时，先完成 DSL、Prompt、测试集和文档，明确阻塞。

## 交付内容

- Workflow/Chatflow DSL；
- Prompt 与输出 Schema；

- 部署、备份和恢复说明；

## 验收标准

- 20 问全部有来源或明确证据不足；
- 主要判断 evidence 覆盖率 100%；
- 无证据推断率达到验收目标；
- JSON/结构化输出成功率 100%；
- 可回溯到课程、案例和时间范围；
- Dify 在线状态、Dataset、索引和应用状态有真实证据。
