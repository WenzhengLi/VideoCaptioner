# 任务执行总览

## Definition of Ready

任务可以领取的条件：

- 所有前置任务已通过验收；
- 公共接口已经存在且有测试；
- 输入和输出模型已冻结；
- 当前没有其他 Agent 修改相同所有权路径。

## Definition of Done

任务完成必须同时满足：

- 任务范围内的功能实现完整；
- 所有“必须交付”文件存在；
- 单元测试覆盖正常路径和至少一个失败路径；
- 静态检查与非集成测试通过；
- 任务文件中的验收项逐条核对；
- 提供交接说明；
- 未修改任务明确列出的非目标内容。

## 数据契约

TASK-001 应冻结以下公共模型，后续 Agent 不得私自改变语义：

- `MediaInfo`：媒体元数据；
- `TranscriptSegment`：FunASR 原始文字区间；
- `SpeakerTurn`：WeSpeaker 原始人物区间；
- `SpeechSegment`：对齐后的人物讲话；
- `BoardCandidate`：检测阶段候选区域；
- `BoardSegment`：稳定课板版本及 OCR；
- `TimelineEntry`：声音和课板的时间轴关联；
- `AnalysisResult`：最终结果。

所有区间均使用整数毫秒和 `[start_ms, end_ms)` 语义。

## 并行开发边界

TASK-002、003、005 可以并行，但只能依赖 TASK-001 已冻结的接口。TASK-004、006、007 可以并行。TASK-008 是第一次允许同时修改顶层流水线编排的任务。

## 测试层级

- 单元测试：使用 mock/fake，不加载真实模型；
- 组件集成测试：本地模型可用时执行，标记 `integration`；
- 端到端测试：由 TASK-010 管理，不属于单个适配器任务；
- 手动视觉验收：保存调试图到临时任务目录，不提交生成物。

```text

开工前必须阅读：
- docs/00-总体方案.md
- docs/01-任务拆分.md
- docs/02-Agent协作与交付规范.md
- docs/tasks/README.md
- 目标任务文件

只修改任务“允许修改”中声明的路径。完成全部“必须交付”和“验收标准”，运行任务中的验收命令，并按协作规范提供 TASK-XXX 交接说明。若前置依赖、公共接口或测试夹具缺失，停止扩大实现范围，明确报告阻塞项。
```
