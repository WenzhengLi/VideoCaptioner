## Flow 执行方式

建立并持续维护以下 Flow，不得跳过前置依赖：

```text
阶段 A：TASK-000
阶段 B：TASK-001
阶段 C：TASK-002、TASK-003、TASK-005
阶段 D：TASK-004、TASK-006、TASK-007
阶段 E：TASK-008
阶段 F：TASK-009、TASK-010
阶段 G：全量审查与回归测试
```

即使 Cursor Flow 支持并发，也只能在任务索引允许并行时并行。共享工作区内不要让两个执行单元同时修改同一文件。

每完成一个任务都要：

1. 逐条检查任务文件中的“必须完成”和“必须交付”；
2. 运行该任务的验收命令；
3. 修复失败，不能把失败留给后续任务；
4. 在 `docs/tasks/STATUS.md` 中记录状态、修改文件、测试结果和已知限制；
5. 确认下游接口已经稳定，再进入下一阶段。

## 强制技术约束

- 使用 Python 3.11；
- FunASR 只负责转录；
- WeSpeaker 是默认说话人分离引擎；
- CAM++ 仅作为备用；
- FunASR 与 WeSpeaker 独立输出，通过时间区间对齐；
- OpenCV 负责课板候选、追踪、换位与去重；
- PaddleOCR 负责课板文字识别；
- Gradio 提供第一版本地 Web；
- 第一版核心处理不得调用 LLM；
- 重型依赖延迟导入；
- 单元测试不得下载模型或访问网络；
- 真实模型测试使用 `integration` 标记；
- 所有时间使用整数毫秒和 `[start_ms, end_ms)`；
- 原始识别结果与人工修订必须分开保存；
- 核心模块不得依赖 Gradio。
