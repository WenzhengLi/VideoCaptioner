# Dify 产品目标

## 产品定位

新 Dify 应用是一套聊天分析与回复教练。它不从外部理论评价课程，而是复现讲师观察聊天、判断女生心理、选择互动方向和组织语言的方式。

旧 Dify 应用和旧知识库仅用于查找原文、核对案例和验证效果，不继续扩建。新应用、新知识库和新工作流单独建立。

## 用户输入

用户可以输入：

- 女生说的一句话；
- 一段连续聊天记录；
- 当前关系、认识时长和聊天阶段；
- 用户想要达到的沟通方向。

系统需要识别口语近义表达。例如“你在干什么”“在做什么”“干嘛呢”“忙什么”“在吗”应能联想到相近的互动意图，再结合上下文区分具体含义。

## 核心回答结构

每次回答按下面的顺序组织：

1. **她为什么这样说**：先解释字面之下的主要动因；
2. **判断依据**：结合前后文、回复速度、信息量、主动程度和当前阶段说明依据；
3. **其他可能含义**：给出课程体系内成立的其他心理方向，以及分别需要观察什么信号；
4. **三种说话方向**：例如拉近、轻松化解、提供筹码、框架转换、后撤或继续推进；具体方向由课程内容决定，不预设固定标签；
5. **具体回复示例**：每个方向提供可以直接参考的自然语言，不只给抽象原则；
6. **下一轮怎么接**：分别说明女生积极回应、继续顾虑、转移话题或降低投入后怎么继续。

## 要复现的能力

- 从女生一句话反推她可能在确认什么、担心什么、测试什么或期待什么；
- 识别窗口、黄灯、顾虑、SD/ASD、筹码、框架、拉近、后撤、推远、服从与行动信号；
- 解释男生为什么在这个节点这样说，而不是只摘录一句话术；
- 保留完整动作链：上一句导致什么判断，这个判断导致什么回复，这个回复又如何进入下一步；
- 针对同一情境提供多个说话方向，并模仿讲师的表达逻辑和语言风格；
- 根据用户提供的上下文调整回复强度，而不是机械匹配关键词。

## 当前阶段不做的内容

- 不把旧 P04 结构直接搬入新知识库；
- 不提前设计完整概念体系；
- 不在 C019 内容和案例链确认前导入 Dify；
- 不混入未来的反制、揭露、博客或视频视角；
- 不使用外部心理学体系修正或质疑讲师的说法。

## 建设顺序

1. 用户确认 C019 内容地图是否准确抓住课程；
2. 从 C019 提炼“原话—讲师判断—动作目的—具体说法—下一步”的知识单元；
3. 用真实用户问题测试多含义分析和三方向回复；
4. 根据测试结果确定新知识库字段、切片方式和检索扩展；
5. 最后新建 Dify 应用、知识库和工作流。
