## <a id="ob-c010-001"></a>OB-C010-001：评论成都疫情后的切私聊
- 标签：[真实性测试](../../课程场景标签.md#tag-scene-sincerity-test)、[需要安全感](../../课程场景标签.md#tag-psych-safety)、[推拉](../../课程场景标签.md#tag-action-push-pull)、[坏坏](../../课程场景标签.md#tag-tone-bad)
- course_chain：`成都那边有疫情吗？` → 她主动评论成都相关朋友圈，说明成都或发圈人进入了她的注意范围；讲师把评论视为老资源重新打开的入口。 → 不在评论区长聊，切私聊说明成都情况，并顺势问她是否想来玩。
- supplement_chain：课程只给出一次直切，未展开她只是随口问、真的想去、或想重新联系时的不同接法。 → 把低成本评论变成可判断投入的私聊 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-002"></a>OB-C010-002：没有闲情雅致后的继续推进
- 标签：[未来场景](../../课程场景标签.md#tag-scene-future-picture)、[邀约](../../课程场景标签.md#tag-action-invite)、[窗口](../../课程场景标签.md#tag-concept-window)、[直球](../../课程场景标签.md#tag-action-direct)
- course_chain：`我没有那个闲情雅致。` → 表面否定旅游，但仍持续回复；讲师没有把它当红灯，而是继续补充自己在成都定居、也会去佛山谈项目的现实交集。 → 用工作和城市交集降低异地感，再继续了解她的时间和可行动性。
- supplement_chain：缺少对真忙、客气后撤和等男方给理由三种情况的区分。 → 不硬顶拒绝，同时制造现实交集 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-003"></a>OB-C010-003：等你回来有机会了解一下
- 标签：[窗口](../../课程场景标签.md#tag-concept-window)、[邀约](../../课程场景标签.md#tag-action-invite)、[直球](../../课程场景标签.md#tag-action-direct)、[坏坏](../../课程场景标签.md#tag-tone-bad)
- course_chain：`等你回来可以有机会了解一下。` → 讲师认为“有机会”遥遥无期，真正的机会需要男人主动把握并落到时间。 → 从抽象机会推进到具体时间，并提出订票来成都两天的新体验。
- supplement_chain：课程没有展示不同窗口大小下，应该直接定期、先通话还是后撤观察。 → 把礼貌可能性变成可执行节点 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-004"></a>OB-C010-004：为什么不是你飞回来见我
- 标签：[真实性测试](../../课程场景标签.md#tag-scene-sincerity-test)、[需要安全感](../../课程场景标签.md#tag-psych-safety)、[推拉](../../课程场景标签.md#tag-action-push-pull)、[坏坏](../../课程场景标签.md#tag-tone-bad)
- course_chain：`你飞回来见我不行吗？` → 她要求男方先投入，本质是不确定独自去陌生城市是否值得，也在确认男方会不会照顾她。 → 给出公司离不开的现实理由，同时承诺她来后可以陪伴、照顾和共同体验。
- supplement_chain：需补足她在测试诚意、担心安全和单纯嫌麻烦时的差异。 → 解释方向而不陷入谁更应该付出的争执 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-005"></a>OB-C010-005：怕遇到下头男或被睡后消失
- 标签：[未来场景](../../课程场景标签.md#tag-scene-future-picture)、[邀约](../../课程场景标签.md#tag-action-invite)、[窗口](../../课程场景标签.md#tag-concept-window)、[直球](../../课程场景标签.md#tag-action-direct)
- course_chain：`女生去陌生城市，怕遇到下头男，也怕喜欢上后对方睡完就走。` → 讲师把远程奔赴的核心顾虑归纳为两种：本人不匹配，或投入感情后被短期处理。 → 通过语音、现实信息、长期价值观和落地安排提高确定性。
- supplement_chain：课程未把这两个顾虑拆成可检索的上下文信号和逐项回应。 → 把抽象不安全拆成可验证问题 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-006"></a>OB-C010-006：星座属相不合的价值观测试
- 标签：[窗口](../../课程场景标签.md#tag-concept-window)、[邀约](../../课程场景标签.md#tag-action-invite)、[直球](../../课程场景标签.md#tag-action-direct)、[坏坏](../../课程场景标签.md#tag-tone-bad)
- course_chain：`属牛和属兔好像不太合。` → 讲师认为直接嘲笑迷信是在挑战她投入已久的价值观；应顺着她的体系合理化，并抓住“绝配”信息升高关系。 → 不否定她相信的东西，用不相冲、可以包容和其他合盘解释保留操作空间。
- supplement_chain：需要补足顺势、轻调侃、反证和把判断交回真实相处四种回复。 → 尊重她的判断体系，同时不被一句标签判死 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-007"></a>OB-C010-007：你是找女朋友还是奔结婚
- 标签：[真实性测试](../../课程场景标签.md#tag-scene-sincerity-test)、[需要安全感](../../课程场景标签.md#tag-psych-safety)、[推拉](../../课程场景标签.md#tag-action-push-pull)、[坏坏](../../课程场景标签.md#tag-tone-bad)
- course_chain：`你是想找女朋友，还是奔着结婚去的？` → 她在确认长期方向；讲师认可“顺其自然的顺序”，并指出这种回答也暗含不立即承诺结婚。 → 承认恋爱到结婚的顺序，强调看真实相处而不是先签结果。
- supplement_chain：需要补足不同认真程度下的透明说法，避免既画饼又含糊。 → 表达认真但不做超出关系阶段的承诺 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-008"></a>OB-C010-008：想睡我与住一起的质问
- 标签：[未来场景](../../课程场景标签.md#tag-scene-future-picture)、[邀约](../../课程场景标签.md#tag-action-invite)、[窗口](../../课程场景标签.md#tag-concept-window)、[直球](../../课程场景标签.md#tag-action-direct)
- course_chain：`你想睡我？` → 在已建立长期和异地奔赴预期后，讲师把它视为打情骂俏式明牌，不主张突然认怂；但亲密必须建立在双方自愿和真实感觉上。 → 承认吸引，强调自然发生、双方都想和不把见面缩成单一结果。
- supplement_chain：课程缺少不同窗口下，从直接承认到降速确认的梯度。 → 保持男女张力，同时把选择权说清楚 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-009"></a>OB-C010-009：大姨妈试探与三个点后撤
- 标签：[窗口](../../课程场景标签.md#tag-concept-window)、[邀约](../../课程场景标签.md#tag-action-invite)、[直球](../../课程场景标签.md#tag-action-direct)、[坏坏](../../课程场景标签.md#tag-tone-bad)
- course_chain：`我来大姨妈去找你，可以吗？` → 讲师认为她在阴阳怪气试探男方是否只为性；三个点的短后撤不给她继续架框，随后再表达无论生理期都欢迎她来。 → 先用低反应打断挑衅，再回到“来是因为想见你”的关系理由。
- supplement_chain：需补足温柔、坏坏、直球和真顾虑识别。 → 证明邀约不依赖单一亲密结果 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。

## <a id="ob-c010-010"></a>OB-C010-010：订票与错误中转的执行复盘
- 标签：[真实性测试](../../课程场景标签.md#tag-scene-sincerity-test)、[需要安全感](../../课程场景标签.md#tag-psych-safety)、[推拉](../../课程场景标签.md#tag-action-push-pull)、[坏坏](../../课程场景标签.md#tag-tone-bad)
- course_chain：`话说你这定的啥，在机场过一天。` → 课程后段明确暴露执行错误：佛山出发经上海长时间中转，不如从广州白云直飞；聊天推进不能代替行程常识。 → 承认安排不合理，重新比较机场、交通、总耗时和改签成本。
- supplement_chain：需要形成 Dify 可回答的落地执行清单，而不是只保留话术。 → 用修正行动保护已建立的信任 → 四种策略与后续分支。
- 状态：`pending_user_confirmation`。
