# Chat Coach Workspace

这是新的聊天分析知识库工作区。旧的课程案例/Dify 数据只作为原文与证据参考，不再决定本工作区的知识结构。

## 当前目标

先完整分析一节课程实际讲了什么，再从课程内容中归纳讲师如何判断心理动因、识别聊天信号、安排互动动作，以及如何组织具体回复。

首个试点课程：`C019`。

## 工作顺序

1. 开放式课程内容地图；
2. 案例对话与状态变化；
3. 课程概念和讲师解释；
4. 可复用信号与动作；
5. 回复策略与示例；
6. 用户问题测试；
7. 最后才设计 Dify 知识单元和检索流程。

## 分析原则

- 不提前假设课程必须包含 ASD、推拉或服从性测试；
- 不把课程事实、讲师解释和我们的推断混写；
- 不使用外部心理理论改写课程，也不从反制或批判视角评价讲师；
- 课程出现什么就记录什么，讲师如何解释就先准确还原其内部逻辑；
- 当前产品只研究讲师如何分析对方、如何推进互动和如何说话；反制体系以后单独建设；
- 不以旧 P04 的案例摘要作为最终结构；
- 不在用户审核内容地图前导入 Dify。
