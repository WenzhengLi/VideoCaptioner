# C002 候选补足稿

状态：第一版 OB 双链候选补足，全部等待用户确认。

本课重点是“窗口—信任—执行—胜负心”的变化。课程事实与 Codex 候选补足严格分开；每个对象提供四个真实不同方向，每方向三条表达。

## SUP-C002-001：她添加后主动打招呼

- 对象编号：`OB-C002-001`
- 场景标签：`stage:破冰`、`signal:窗口上升`、`signal:主动开启`、`psychology:愿意认识`、`action:测试`、`action:分享`、`tone:轻松`、`concept:窗口`
- 课程原链 `course_chain`：女生添加后主动招呼 → 讲师认为存在初始窗口 → 男生不急着做长篇展示 → 进入爱情状态和主动创造故事的话题。
- 补足策略链 `supplement_chain`：先区分礼貌招呼、主动好奇和明显热情 → 用状态交换、具体钩子、坏坏奖励或轻邀约测试她愿意投入多少。
- 具体回复示例：
  - 方向 A：自然状态交换。适合只有一句招呼。
    1. “刚忙完，你今天在什么状态？”
    2. “嗨，今天是工作模式还是偷懒模式？”
    3. “来得正好，我刚想找个人聊点轻松的。”
  - 方向 B：接她朋友圈细节。适合有可观察内容。
    1. “你刚发的那家店看着不错，真的好吃还是只适合拍照？”
    2. “你照片里的小浣熊干脆面，把我小时候记忆勾出来了。”
    3. “你最近一直在跑活动，体力比我想的好。”
  - 方向 C：坏坏奖励主动。
    1. “会主动打招呼，加分，暂时不把你归到高冷组。”
    2. “开场挺乖，接下来看看聊天能力。”
    3. “你先开口了，那我负责把故事继续下去。”
  - 方向 D：顺势做小测试。适合回复速度和情绪都高。
    1. “现在有空聊五分钟，还是只能路过一下？”
    2. “你在附近的话，晚点可以喝一杯。”
    3. “先告诉我你在哪一片，我判断我们是不是近邻。”
- 对方不同反应后的下一步：她反问男生状态就进入分享；她接具体细节就围绕共同记忆展开；只礼貌回复则不直接判断为大窗口；主动报位置或时间才进入邀约。
- 来源边界：主动招呼被视为初始窗口来自课程；四种测试路径为候选补足。
- 确认状态：待用户确认

## SUP-C002-008：“你在钓鱼，你在骗我”

- 对象编号：`OB-C002-008`
- 场景标签：`stage:邀约落地`、`scene:真实性测试`、`scene:误会指控`、`psychology:确认认真程度`、`action:事实解释`、`action:行动证明`、`tone:强势`、`concept:框架`
- 课程原链 `course_chain`：女生根据男生失联推断他在陪其他女人、钓鱼和撒谎 → 讲师要求给详细逻辑和事实 → 不能只说“你误会了” → 用行动恢复可信度。
- 补足策略链 `supplement_chain`：先分清她质疑的是事实、优先级还是失望 → 选择时间线解释、可核验行动、短电话或边界表达 → 解释后不无限自证。
- 具体回复示例：
  - 方向 A：完整事实链。
    1. “我几点去处理什么、几点结束、为什么没及时回，现在按顺序告诉你。”
    2. “你看到的是我消失，缺的那段信息是我在处理这件事，不是陪别人。”
    3. “我没提前同步是问题，但‘有其他女人’不是事实。”
  - 方向 B：用行动证明。
    1. “不靠文字争，我现在打给你。”
    2. “我把到达时间发你，到了再判断我是不是只会说。”
    3. “今晚做不到我就明确取消，不再用‘马上’拖你。”
  - 方向 C：接住她被轻视的感觉。
    1. “你真正生气的不是证据，而是你觉得自己被排在最后。”
    2. “你已经为见面做准备，我却让你一直猜，这会让人觉得不被重视。”
    3. “我先修复优先级问题，再谈你对我的判断。”
  - 方向 D：强势停止无限审讯。
    1. “该解释的我会解释，但不存在的事我不会认。”
    2. “你可以根据行动判断我，不用让我为每个猜测证明一次。”
    3. “事实说完后，看我们还要不要见，不继续互相定罪。”
- 对方不同反应后的下一步：她接受事实就进入执行；继续追问可再补一次关键证据；话题变成人格羞辱则结束争论；她要求明确时间就给能兑现的方案。
- 来源边界：详细逻辑解释和不承认欺骗来自课程；四方向为候选补足。
- 确认状态：待用户确认

## SUP-C002-009：被质疑时保留态度，不崩框架

- 对象编号：`OB-C002-009`
- 场景标签：`stage:暧昧升温`、`psychology:观察男生框架`、`signal:黄灯`、`action:表达态度`、`action:推拉`、`action:行动证明`、`tone:强势`、`concept:框架`
- 课程原链 `course_chain`：女生激烈质疑 → 男生说“没有就是没有，我不会欺骗任何人” → 讲师认为这是案例最好的一段 → 态度不等于拒绝承担失联问题。
- 补足策略链 `supplement_chain`：把“事实边界”和“行为责任”拆开 → 对没有发生的事不认，对确实造成的失望承担 → 再把关系带回行动。
- 具体回复示例：
  - 方向 A：事实边界。
    1. “我没有骗你，这一点不会因为你生气就改口。”
    2. “你可以质疑安排，但不要把猜测当结论。”
    3. “没有其他人就是没有，我不会编故事哄你。”
  - 方向 B：承担实际问题。
    1. “我没骗你，但我确实没把进度告诉你。”
    2. “事实我坚持，失联造成的失望我承担。”
    3. “不是背叛，是执行很差，这两个问题别混在一起。”
  - 方向 C：坏坏拉回关系。
    1. “你这么凶，说明已经把我管进你的范围了？”
    2. “审我可以，审完要给我一个见面改判的机会。”
    3. “醋味我闻到了，案情我也解释完了。”
  - 方向 D：用下一步结束争论。
    1. “现在两个选择：电话说清，或者明天见面说清。”
    2. “我不继续刷屏自证，给你一个准确到达时间。”
    3. “判断我看后面的行动，不看谁在聊天框里更凶。”
- 对方不同反应后的下一步：她被调侃后缓和可转电话；仍强烈不信任就不用坏坏路线；接受责任后必须实际兑现；若不能兑现，停止使用强势话术掩盖问题。
- 来源边界：核心态度句和讲师评价来自课程；拆分模型为候选补足。
- 确认状态：待用户确认

## SUP-C002-010：删除、重加与关系修复

- 对象编号：`OB-C002-010`
- 场景标签：`stage:关系升级`、`scene:删除重加`、`signal:持续回流`、`psychology:愤怒但未完全退出`、`action:修复`、`action:后撤`、`tone:坏坏`、`concept:窗口`
- 课程原链 `course_chain`：女生多次因失望删除 → 男生再次申请或解释后她又加回 → 课程把回流视为仍有窗口，同时把反复删除视为信任已被严重消耗。
- 补足策略链 `supplement_chain`：重加后不假装无事，也不长篇清算 → 简短确认问题、提出新规则、给一次行动修复或在模式重复时后撤。
- 具体回复示例：
  - 方向 A：简短承认后重启。
    1. “加回来就把上次的问题处理掉：我不再让你无期限等。”
    2. “前面节奏很乱，这次从一个能兑现的安排开始。”
    3. “不翻旧账，但旧问题要有新做法。”
  - 方向 B：建立沟通规则。
    1. “有变化我提前说，你生气也直接说，别靠删除传话。”
    2. “可以暂停聊天，不用每次把关系清空。”
    3. “我负责同步，你负责别突然消失，我们各改一半。”
  - 方向 C：坏坏重开。
    1. “第四次好友申请通过，会员关系有点曲折。”
    2. “删人速度挺快，舍不得的速度也不慢。”
    3. “这次先别碰删除键，给真人见面一次机会。”
  - 方向 D：模式重复时后撤。
    1. “如果还靠删加解决问题，我们先停一停。”
    2. “我愿意修复一次，不愿意无限重演。”
    3. “等双方都能稳定沟通时再继续。”
- 对方不同反应后的下一步：她愿意定规则就安排见面；只想清算则先听核心诉求；重加后仍不给任何行动，不把它自动判断为高窗口；再次删除则不连环申请。
- 来源边界：多次删除重加和窗口判断来自课程；修复规则为候选补足。
- 确认状态：待用户确认

## SUP-C002-011：“我来例假了”在本案例中的含义

- 对象编号：`OB-C002-011`
- 场景标签：`stage:邀约落地`、`scene:身体状态顾虑`、`signal:黄灯`、`psychology:保险栓与测试`、`action:框架转换`、`action:给台阶`、`tone:温柔`、`concept:SD-ASD`
- 课程原链 `course_chain`：经历两次严重失望后，女生说来例假 → 讲师解释为带报复意味的测试，也是一枚保险栓 → 她可以继续见面，但不提前承诺任何结果 → 男生说很多事情不必一晚上完成。
- 补足策略链 `supplement_chain`：只能结合本案前文判断，不能把所有经期表达一律定义为测试 → 接受身体状态，解除结果压力，再观察她是否继续给见面行动。
- 具体回复示例：
  - 方向 A：解除结果压力。
    1. “没关系，见面不是为了一个晚上完成什么。”
    2. “先吃东西、聊天，后面看你状态。”
    3. “你不用用结果证明想见我，来见到就够了。”
  - 方向 B：照顾身体状态。
    1. “那别喝太刺激的，我给你准备热的。”
    2. “不舒服就把安排弄轻一点，我过去找你也可以。”
    3. “先告诉我你现在难不难受，方案按身体状态改。”
  - 方向 C：坏坏识别保险栓。适合她语气轻松、仍主动推进。
    1. “保险声明收到，今晚先负责见面，不提前验收结果。”
    2. “你把免责条款写在前面了，看来还是想来。”
    3. “放心，我要的是你出现，不是让你交作业。”
  - 方向 D：回到行动验证。
    1. “那还见吗？见就定吃什么和几点。”
    2. “身体允许就来，不允许就换明天，不用绕。”
    3. “是否有窗口看你后面是否继续给方案，不靠这句话单独判断。”
- 对方不同反应后的下一步：她继续问吃什么说明见面链仍在；真不舒服就延期；明确只是告知，不追加含义；她说不见就停止推进。
- 来源边界：报复性测试、保险栓和“不必一晚做完”来自课程；普适限制和四方向为候选补足。
- 确认状态：待用户确认

## SUP-C002-012：她提出“叫外卖”

- 对象编号：`OB-C002-012`

- 课程原链 `course_chain`：女生确认家里是否有吃的 → 主动提出叫外卖 → 课程把外卖看作她去家里的合理步骤和仪式 → 从抽象见面进入可执行生活场景。
- 补足策略链 `supplement_chain`：奖励她提出方案 → 继续确认口味、地址、到达顺序和备选公共方案 → 不把点外卖自动理解为同意其他结果。
- 具体回复示例：
  - 方向 A：奖励行动配合。
    1. “这个方案靠谱，你终于开始一起做计划了。”
    2. “外卖你挑，我负责把地方和饮料准备好。”
    3. “会解决问题的人加分，想吃什么？”
  - 方向 B：进入执行细节。
    1. “你几点到？我按时间下单。”
    2. “地址我发你，外卖别太早凉掉。”
    3. “你负责选口味，我确认家里还有什么。”
  - 方向 C：坏坏共同生活画面。
    1. “还没见面就开始安排家里吃什么，进入状态挺快。”
    2. “外卖可以，第一口归我，算场地费。”
    3. “你带人来，我让外卖负责暖场。”
  - 方向 D：给可退台阶。
    1. “如果临时觉得去家里快了，就在附近店里吃。”
    2. “先外面碰头，再决定外卖送哪。”
    3. “只锁定吃饭，后面按现场感觉。”
- 对方不同反应后的下一步：她给口味和到达时间就执行；仍担心地点就切公共方案；提出自己下单可让她参与；明确只吃饭就尊重边界，不额外解读。
- 来源边界：外卖作为到家步骤来自课程；执行细节和边界为候选补足。

## SUP-C002-013：高窗口被消耗后，喜欢叠加胜负心

- 对象编号：`OB-C002-013`
- 场景标签：`stage:关系升级`、`scene:反复失约`、`signal:持续追发`、`psychology:胜负心与面子修复`、`action:诊断`、`action:修复`、`action:后撤`、`concept:窗口`
- 课程原链 `course_chain`：女生原本高窗口 → 被连续放鸽子、等待和删除重加 → 仍执着完成见面 → 讲师判断后期心态包含“这个男的必须拿下” → 最终结果不等于关系质量仍然良好。
- 补足策略链 `supplement_chain`：区分喜欢、愤怒投入、竞争感和面子修复 → 不继续故意制造稀缺 → 用稳定兑现测试她是否愿意在没有追逐刺激时正常相处。
- 具体回复示例：
  - 方向 A：点明互动已经变味。
    1. “我们现在像在争谁先输，不像在好好见面。”
    2. “你想见我是真的，想证明自己没被耍也是真的。”
    3. “先把输赢拿掉，看我们还想不想相处。”
  - 方向 B：稳定兑现修复。
    1. “这次只定一个我一定做到的时间，别再靠拉扯维持情绪。”
    2. “我按时到，你不用追；你愿意见，就正常出现。”
    3. “用一次平稳见面替代下一轮删加。”
  - 方向 C：坏坏降竞争感。
    1. “不用非得拿下我，先看看真人值不值得你这么较劲。”
    2. “胜负欲先收一半，喜欢留一半。”
    3. “你已经证明够执着了，接下来让我证明能不能靠谱。”
  - 方向 D：无法恢复稳定时后撤。
    1. “如果我们只剩下谁折磨谁，先停。”
    2. “我不再用忽冷忽热换投入，你也不用靠追逐证明自己。”
    3. “等情绪落下来，再决定是否继续。”
- 对方不同反应后的下一步：平稳相处后仍有兴趣，说明不只靠胜负心；一旦没有追逐就失去投入，关系动力可能主要来自竞争；再次失约会加重问题；她只想报复或羞辱则结束。
- 来源边界：“必须拿下”的课程判断来自原课；区分和稳定验证为候选补足。
- 确认状态：待用户确认

## SUP-C002-014：工作讨论、真实圈层与同行感

- 对象编号：`OB-C002-014`
- 场景标签：`stage:建立吸引`、`scene:工作讨论`、`scene:圈层证明`、`psychology:确认真实价值`、`action:分享`、`action:提供筹码`、`action:框架转换`、`concept:吸引力`
- 课程原链 `course_chain`：双方讨论抖音电商 → 对 IP 和供应链逻辑产生争论 → 男生展示朋友圈里真实的带货成绩 → 女生看到圈层、能力和潜在合作价值 → 没有大量情感话题也能继续相处。
- 补足策略链 `supplement_chain`：先真诚交换行业判断 → 用具体案例而非抽象吹嘘证明 → 允许有分歧 → 把交流转成共同研究、资源互换、轻邀约或带男女感的同行欣赏。
- 具体回复示例：
  - 方向 A：专业好奇。
    1. “你说货带人，我想听你拆一个最近真正跑出来的案例。”
    2. “你今年看到的平台变化里，哪一个最影响利润？”
    3. “别给结论，给我讲讲你是怎么判断供应链的。”
  - 方向 B：拿真实证据交流。
    1. “我朋友圈刚好有个直播间在线三万的案例，给你看数据，不只聊概念。”
    2. “我们团队做 IP 的实际转化是这样，你看和你的逻辑哪里冲突。”
    3. “我认识的人正在做这个品类，可以用真实结果对一下。”
  - 方向 C：把分歧变成吸引。
    1. “你敢当面反驳我这点挺吸引，比只会附和有意思。”
    2. “今天先争出个高下，下一顿继续拿数据说话。”
    3. “观点不一样没关系，脑子转得快的人我愿意多聊。”
  - 方向 D：从同行感落到见面。
    1. “这话题微信说不完，吃饭时各带一个案例。”
    2. “你来选店，我带数据，我们做一场不太正经的行业复盘。”
